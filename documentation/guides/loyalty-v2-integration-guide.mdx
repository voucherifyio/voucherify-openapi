To integrate Voucherify's loyalty v2 with your platform, you'll need to fulfil several steps. You'll have to prepare your customer data, build the loyalty program with required modules, and create the loyalty member journey.

Loyalty v2 has a modular structure, so you can start small and scale your program up as it grows with your members. The modular structure also helps you to optimize your loyalty program with your other incentives, like discount coupons or gift cards.

## Preparation: Technical implementation checklist

Prepare your Voucherify project for a functional loyalty v2 integration.

### Environment readiness (The Dashboard)

[ ] Create a Dedicated Project: Ensure you are working in a Test/Sandbox project. v2 BETA features should be isolated from your production v1 loyalty programs.

[ ] Configure a Loyalty v2 Program:

Create an architecture of what you need from the program: required vs additional modules. Start small, experiment,

Navigate to Campaigns > Loyalty Programs.

Ensure the program type is v2 (Program-based).

Set the program status to ACTIVE.

[ ] Define Card Assignments: In the program settings, verify that at least one Card Definition is assigned. (In v2, members cannot exist without at least one assigned loyalty card definition).

[ ] Set Up Earning Rules: Define at least one "Manual" or "Custom Event" earning rule to test point allocation.

### Authentication and Security

`Possibly to be removed`

[ ] Retrieve API Keys: Collect your X-App-Id and X-App-Token from Project Settings > Application Keys.

[ ] Identify Program ID: Capture the lprg_... ID of your v2 program. This is required for all member-related API calls.

[ ] Check Permissions: Ensure your API key has "Write" access for the loyalties domain.

### Data preparation

[ ] Customer Sync Strategy: Decide if you will use existing cust_... IDs or create new customers on-the-fly during member enrollment.

[ ] Metadata Schema: If you plan to use custom metadata for Members or Cards, ensure the keys are defined in your Metadata Schema settings to avoid validation errors.

[ ] Custom events

[ ] Currency Alignment: Verify that your application handles loyalty points as integers. (Reminder: 1 point = 1 cent in the default cashback logic).

## Acquisition

[ ] Enroll Member: Call POST /api/v2/loyalties/programs/{programId}/members and verify the lmbr_... ID is returned.

ACTIVE/INACTIVE

[ ] Verify Card Generation: Confirm the response contains a cards array with at least one lcrd_... ID.

[ ] Handle Card Code Latency: Ensure your UI can handle a null or "Generating" state for card codes, as v2 card codes are generated asynchronously after member creation.
