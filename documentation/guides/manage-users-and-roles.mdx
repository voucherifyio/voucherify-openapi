Voucherify uses **users**, **projects**, and **roles** to control access to data and actions. This setup keeps access structured and permissions clear.

## Key concepts

<AccordionGroup>

<Accordion title="Owner">
The **owner** is not a role but a special flag assigned to a user account.

The owner flag:
- Grants access to **Team Settings**
- Allows managing team-level settings such as users, roles, billing, and security

The owner flag does **not** change role permissions. An owner still operates within the permissions of their assigned role (e.g., Viewer or User).

**How to become an owner**
- Owner status can be granted only by an existing owner
- If your team does not have an active owner, contact Voucherify support
</Accordion>

<Accordion title="Users">
**Users** are people invited to work in the Voucherify dashboard.

To send an invitation:
- A user must be assigned to at least one project

After accepting the invitation:
- The user receives an email with a link to set a password and log in
- The user gains access only to the assigned projects
</Accordion>

<Accordion title="Projects">
**Projects** are separate workspaces within Voucherify that contain campaigns, customers, and related data.

Access is managed per project, which means:
- A user can have access to selected projects only
- A user’s role can differ between projects
</Accordion>

<Accordion title="Roles">
**Roles** define what actions a user can perform, such as viewing data, creating campaigns, or managing settings.

Custom roles are defined at the organization level, while users are assigned roles within projects.
</Accordion>

</AccordionGroup>

## Invite a new team member

Only the **account owner** can invite new users.

<Steps>
  <Step title="Open Team Settings">
    Select your username in the bottom-left corner and go to **Team Settings**.
  </Step>

  <Step title="Start the invitation">
    Open the **Team** tab and choose **Invite member**.
  </Step>

  <Step title="Provide user details">
    Enter the user’s first name, last name, and email address.
  </Step>

  <Step title="Assign projects and roles">
    Select at least one project and assign a **role** within that project (required to send the invitation).
  </Step>

  <Step title="Send the invitation">
    Confirm with **Invite Member**. The user receives an email to activate their account.
  </Step>
</Steps>

After activation, the user appears in the team list. You can later update their roles or project access.

## User roles

### Built-in roles

Voucherify includes the following predefined roles:

| Role | Description |
| --- | --- |
| **Admin** | Full access to all data and configuration within assigned projects. |
| **User** | Can create, edit, and export data in assigned projects but cannot manage users or projects. |
| **Viewer** | Can view and export data but cannot make changes. |
| **Restricted user** | Can work only within assigned areas or stores, defined by an Admin or owner. |
| **Merchant** | Has limited dashboard access and can validate and redeem single incentives. |

### Custom roles (Enterprise)

With an Enterprise plan, Admin users can create **custom roles** with specific permissions.

To create a custom role:
1. Go to **Team Settings** > **Roles**
2. Duplicate an existing role
3. Enable or disable selected permissions
4. Save the role

You can then assign the custom role to users within selected projects.

## Manage user access

You can manage users from **Team Settings** > **Team**.

From the user list, you can:
- Add a user to a project
- Change the user’s role in a project
- Remove the user from a project
- Update user details

**Who can manage users**
- Only users with access to **Team Settings** (owners) can manage users and roles

## Access for partners and affiliates (Merchants)

If partners do not need full dashboard access, you can provide **API-based access with limited permissions**.

Merchants:
- Can access the dashboard with limited permissions
- Can validate and redeem single incentives
- Cannot use stackable redemptions
- Can only use API keys assigned to them (like any integration)

### Create merchant API access

To create **merchant API access**:
1. Go to **Project Settings** > **General** > **Application Keys**
2. Create a new API key
3. Assign the **Merchant** role
4. Share the key securely with the partner

<Info>
**Management API (Enterprise)**  
You can manage users, roles, and project assignments programmatically using the Management API.
</Info>
