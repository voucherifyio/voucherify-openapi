The dynamic formula builder defines how discount or loyalty point values are calculated.

Instead of using a static value for discounts or loyalty points, you can create formulas using mathematical and logical operators combined with metadata values. The final value is calculated dynamically during redemption.

The formula builder requires a **fallback value**. The fallback value is used if the dynamic formula cannot be evaluated due to missing parameters.

You can build formulas using:

- the **Formula builder** interface
- the **Expression output**, which allows direct editing of the formula expression

## Application scope

### Dynamic discount formulas

The dynamic discount formula builder applies to:

- Order discount  
- Product discount  
- Reward  

It can be used when creating:

- coupon campaigns
- in-cart promotions
- referral codes

### Dynamic points formulas

The dynamic points formula builder applies to:

- Fixed
- Proportional earning rules

It can be used when creating **loyalty programs**.

## Context-specific behavior

Available operands and operators depend on the **formula context**, such as:

- discount type
- promotion configuration
- campaign configuration

Some operands or operators may not be available depending on configuration.

## Formula builder

You can use the **Formula builder utility** to construct formulas using a visual interface.

<Steps>
  <Step title="Open formula builder">
    Click the **Formula** button next to the discount value or points.
  </Step>

  <Step title="Build formula">
    Build the formula by selecting an **operand** followed by an **operator**.

    You can also paste a complete expression in **Expression output**.  
    The formula builder automatically converts it into the visual formula structure.

    Click **+** to add elements and **X** to remove them.
  </Step>

  <Step title="Remove operator">
    Click the operator and select **Remove**.
  </Step>

  <Step title="Save">
    Click **Save** after defining the formula.
  </Step>
</Steps>

To edit an existing formula, click the **Formula** button again.

To remove a saved formula, click the **—** button next to the Formula button.

## Operand reference

Operands represent the values used inside dynamic formulas.

### Metadata and base operands

| Operand | Definition | Example |
|---|---|---|
| Number | Numeric value. | `2` |
| Text | String value. | `"VIP"` |
| Order metadata | Order metadata attribute. | `ORDER_METADATA("day_of_week")` |
| Voucher metadata | Voucher metadata attribute. | `REDEEMABLE_METADATA("active_on_calendar_month")` |
| Customer metadata | Customer metadata attribute. | `CUSTOMER_METADATA("customer_life_time_value")` |
| Redemption metadata | Redemption metadata attribute. | `REDEMPTION_METADATA("store_list")` |
| Publication metadata | Publication metadata attribute. | `PUBLICATION_METADATA("year")` |
| Promotion tier metadata | Promotion tier metadata attribute. | `REDEEMABLE_METADATA("membership_duration")` |

### Expression functions

| Operand | Definition | Example |
|---|---|---|
| Brackets | Groups expressions together. | `(10 - ORDER_METADATA("number_of_store_visits"))` |
| Min | Returns lowest value. | `MIN(PUBLICATION_METADATA("year"); CUSTOMER_METADATA("customer_life_time_value"))` |
| Max | Returns highest value. | `MAX(PUBLICATION_METADATA("year"); CUSTOMER_METADATA("customer_life_time_value"))` |
| Power | Raises a number to a power. | `POW(ORDER_METADATA("number_of_store_visits");2)` |
| Round | Rounds to specified precision. | `ROUND(CUSTOMER_METADATA("customer_life_time_value");1)` |
| Floor | Rounds number down. | `FLOOR(CUSTOMER_METADATA("customer_life_time_value");1)` |
| Ceil | Rounds number up. | `CEIL(CUSTOMER_METADATA("customer_life_time_value");1)` |
| If | Conditional expression. | `IF(ORDER_METADATA("number_of_store_visits") > 5;10;3)` |
| Default to | Returns first non-null value. | `DEFAULT_TO(REDEMPTION_METADATA("store_list") / 2;5)` |

</Accordion>

## Operator reference

Operators define how operands interact in formulas.

### Number operators

| Operator | Description | Example |
|---|---|---|
| Add (+) | Adds numbers or expressions. | `CUSTOMER_METADATA("number_of_store_visits") + REDEEMABLE_METADATA("active_on_calendar_month")` |
| Subtract (-) | Subtracts numbers or expressions. | `PUBLICATION_METADATA("year") - CUSTOMER_METADATA("customer_life_time_value")` |
| Multiply (x) | Multiplies numbers or expressions. | `ORDER_METADATA("day_of_week") x 2` |
| Divide (÷) | Divides numbers or expressions. | `CUSTOMER_METADATA("customer_months_active") ÷ 8` |
| Modulo (%) | Returns remainder after division. | `REDEEMABLE_METADATA("active_on_calendar_month") % REDEMPTION_METADATA("allowed_free_units")` |
| Greater than (>) | Comparison operator. | `IF(CUSTOMER_METADATA("number_of_store_visits") > 10;20;3)` |
| Less than (\<) | Comparison operator. | `IF(CUSTOMER_METADATA("number_of_store_visits") < 10;3;20)` |
| Equal to (=) | Equality comparison. | `IF(CUSTOMER_METADATA("number_of_store_visits") = 50;50;0)` |
| In array | Checks value inside list. | `IF(ORDER_UNITS_QUANTITY IN_ARRAY "3,6,9";10;5)` |
| Not in array | Checks value outside list. | `IF(ORDER_UNITS_QUANTITY NOT_IN_ARRAY "1,2,3";10;5)` |

### String operators

| Operator | Description | Example |
|---|---|---|
| Is | Returns true if equal. | `IF(CUSTOMER_METADATA("acquisition_channel") is "Facebook";20;CUSTOMER_METADATA("age"))` |
| Is not | Returns true if not equal. | `IF(CUSTOMER_METADATA("acquisition_channel") isNot "Twitter";10;3)` |
| Contains | Returns true if string contains value. | `IF(CUSTOMER_METADATA("location") contains "California";3;0)` |
| Starts with | Returns true if string starts with value. | `IF(REDEMPTION_METADATA("location_id") startsWith "Jersey_Shoreline_store_";15;0)` |
| Ends with | Returns true if string ends with value. | `IF(PUBLICATION_METADATA("publication_method") endsWith "_sms";50;35)` |
| In array | Checks string inside list. | `IF(ORDER_METADATA("region") IN_ARRAY "EU,EMEA";10;5)` |
| Not in array | Checks string outside list. | `IF(ORDER_METADATA("region") NOT_IN_ARRAY "EU,EMEA";10;5)` |

### Date operators

| Operator | Description | Example |
|---|---|---|
| Days since | Days between date and today. | `DAYS_SINCE(CUSTOMER_METADATA("date_last_purchase"))` |
| Months since | Months between date and today. | `MONTHS_SINCE(CUSTOMER_METADATA("date_subscribed"))` |
| Years since | Years between date and today. | `YEARS_SINCE(CUSTOMER_METADATA("acquisition_date"))` |
| Days until | Days until future date. | `DAYS_UNTIL(VOUCHER_EXPIRATION_DATE)` |
| Months until | Months until future date. | `MONTHS_UNTIL(VOUCHER_EXPIRATION_DATE)` |
| Day of month | Extract day from date. | `DAY_OF_MONTH(CUSTOMER_BIRTHDATE)` |
| Month | Extract month from date. | `MONTH(TODAY)` |
| Year | Extract year from date. | `YEAR(CUSTOMER_BIRTHDATE)` |
| Date | Creates date constant. | `DATE("2023-08-17")` |
| Date time | Creates datetime constant. | `DATE_TIME("2023-08-17T14:05:00.000Z")` |
| Date equality | Checks date equality. | `IF(TODAY DATE_IS DATE("2023-08-22");10;5)` |
| Date inequality | Checks date inequality. | `IF(TODAY DATE_IS_NOT DATE("2023-08-22");10;5)` |
| Is after | Checks if date occurs after another. | `IF(TODAY IS_AFTER DATE("2023-08-22");10;5)` |
| Is before | Checks if date occurs before another. | `IF(NOW IS_BEFORE DATE("2023-08-22");10;5)` |

### Complex expression operators

| Operator | How it works | Use case | Example |
|---|---|---|---|
| And | All conditions must be true. | 20% discount after 10 visits **and** Friday purchase. | `IF(CUSTOMER_METADATA("number_of_store_visits") > 10 AND ORDER_METADATA("day_of_week") = 5;20;3)` |
| Or | At least one condition must be true. | 20% discount after 10 visits **or** Friday purchase. | `IF(CUSTOMER_METADATA("number_of_store_visits") > 10 OR ORDER_METADATA("day_of_week") = 5;20;3)` |

## Nested formulas

Nested formulas allow you to place logical expressions inside other expressions.

Example:

```
IF(ORDER_ITEMS_QUANTITY > 4;IF(ORDER_AMOUNT > 200;25;15);10)
```

Example logic:

- If order has **more than 4 items**
  - and order value **> 200 → 25%**
  - otherwise → **15%**
- Otherwise → **10%**

## Switch function

The `SWITCH` function assigns values based on predefined conditions.

Example:

```
SWITCH(ORDER_METADATA("store_city");
SWITCH_CASE("Boston";10)
SWITCH_CASE("New York";15);
5)
```

Example logic:

- Boston → **10% discount**
- New York → **15% discount**
- All other cities → **5% discount**
