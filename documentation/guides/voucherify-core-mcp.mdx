Voucherify Core MCP lets you connect AI agents to your Voucherify data. You can query campaigns, customers, vouchers, orders, and more using natural language in tools like Claude, Cursor, or Visual Studio Code.

[Model Context Protocol (MCP)](https://modelcontextprotocol.io/) is an open standard with two parts:

- **MCP client** — the application where your AI agent runs (Claude Code, Cursor, Visual Studio Code, Windsurf, etc.)
- **MCP server** — a service that defines what tools the AI can use and what data it can access. You host it locally.

<Note>
Only the local MCP server is supported at this time.

Voucherify is planning a remote MCP server. Contact [Voucherify support](https://www.voucherify.io/contact-support) or your Technical Account Manager if you are interested.
</Note>

## What you can do

After you [set up Voucherify Core MCP](#set-up-voucherify-core-mcp), your AI tools can read data about campaigns, promotion tiers, orders, products, vouchers, customers, and more.

Use cases:

- **AI-powered tooling** — Build tools that pull live data from Voucherify.
- **Complex integrations** — Let your engineers combine Voucherify data with other MCP-compatible services.
- **Analytics with natural language** — Technical marketers can fetch and analyze Voucherify data across multiple sources using plain language prompts.

The MCP server uses read-only endpoints. To prevent accidental changes, creating and updating data is not supported.

<Warning>
API calls made through the Voucherify MCP count toward your billing. Complex queries (like best deals) require more calls. Simple queries (like campaign summary) may need only one. Your MCP client asks for confirmation before each call.
</Warning>

## Set up Voucherify Core MCP

### Prerequisites

- An MCP client (Cursor, Claude Desktop, or Visual Studio Code)
- [UV installed](https://docs.astral.sh/uv/getting-started/) — restart your client after a first-time UV install
- *Recommended:* A dedicated Voucherify server-side app ID and token for MCP use

### Add the configuration

<Steps>
  <Step title="Open your MCP client">
    Open the MCP client you want to use.
  </Step>
  <Step title="Add the configuration snippet">
    Add the following snippet to the `mcp.json` file in your client. The exact location varies by client. See [client-specific setup](#client-specific-setup) for details.

    ```json
    {
      "version": 1,
      "mcpServers": {
        "voucherify-core-mcp": {
          "command": "uvx",
          "args": ["voucherify-core-mcp", "--transport", "stdio"],
          "env": {
            "VOUCHERIFY_APP_ID": "<app id>",
            "VOUCHERIFY_APP_TOKEN": "<app token>",
            "VOUCHERIFY_API_BASE_URL": "https://<clusterId>.api.voucherify.io"
          }
        }
      }
    }
    ```
  </Step>
  <Step title="Add your credentials">
    Copy your Voucherify server-side app ID and token from **Project settings** into the snippet.
  </Step>
  <Step title="Set the API base URL">
    Replace the API base URL with the one for your cluster:
    - Europe: `https://api.voucherify.io`
    - North America: `https://us1.api.voucherify.io`
    - Asia: `https://as1.api.voucherify.io`
  </Step>
  <Step title="Run the connection">
    Start the MCP server connection in your client.
  </Step>
  <Step title="Start a conversation">
    Open a new chat and start querying your Voucherify data.
  </Step>
</Steps>

## Available tools

The Voucherify Core MCP provides the following tools for reading data:

<AccordionGroup>
  <Accordion title="find_customer">
    Returns a customer's status and details such as collected loyalty points, reward eligibility, and profile data. You can look up a customer by email, source ID, or Voucherify ID.
  </Accordion>
  <Accordion title="list_campaigns">
    Returns a list of campaigns. Use it to view active, scheduled, or completed campaigns.
  </Accordion>
  <Accordion title="get_campaign_summary">
    Returns a campaign performance summary, including comparisons with past periods (for example, previous week). Use it to track trends and measure results over time.
  </Accordion>
  <Accordion title="get_promotion_tier">
    Returns the configuration of a promotion tier, such as reward levels or thresholds that determine customer benefits.
  </Accordion>
  <Accordion title="qualifications">
    Checks a customer's eligibility for specific campaigns, promotions, or reward rules. Returns only qualified incentives.
  </Accordion>
  <Accordion title="get_best_deals">
    Returns the top 5 best incentives for a customer based on their current context.

    <Note>
    For the best results, set the Application rule to **Partial** in the Voucherify dashboard under **Redemptions** > **Stacking rules**. Read [Stacking rules](https://support.voucherify.io/article/604-stacking-rules) for more details.
    </Note>
  </Accordion>
  <Accordion title="list_products">
    Returns the product catalog, including attributes like pricing, availability, and categories.
  </Accordion>
  <Accordion title="get_voucher">
    Returns details of a specific voucher, such as code, status, balance, and expiration date.
  </Accordion>
  <Accordion title="estimate_loyalty_points">
    Estimates how many points a customer earns in a loyalty campaign for a specific order. The estimate is based on earning rules with the `order-paid` type.

    Identify the customer by ID, source ID, or email. Provide order details like items (name or ID) and quantity. The tool returns the estimated point total.

    <Info>
    This returns an estimate, not an exact value. If a campaign includes tiers, mappings, and multiple earning rules, the final calculation may differ. A customer may change tiers and earn more or fewer points depending on other factors.
    </Info>
  </Accordion>
</AccordionGroup>

## Best practices

<AccordionGroup>
  <Accordion title="Ask specific questions" defaultOpen={true}>
    - Use precise date ranges (for example, "July 2025 redemptions") instead of vague prompts like "recent redemptions."
    - Name the exact campaign, product category, or data type you need.
    - Avoid broad requests like "all campaigns in the last 3 years." They lead to unclear results.
  </Accordion>
  <Accordion title="Add context when results are unclear">
    If results look wrong, reframe your query. If the AI loops or repeats itself, start a new chat with a more detailed prompt.
  </Accordion>
  <Accordion title="Ask follow-up questions">
    Once you get a useful answer, ask the AI to:

    - Suggest additional insights or next steps.
    - Explain how it reached its conclusions. This helps you refine future prompts.
  </Accordion>
  <Accordion title="Try a different model">
    If answers are not useful, switch to a different AI model. Each model has different strengths and is suited for different tasks.
  </Accordion>
</AccordionGroup>

### Prompt examples

Use these prompts as a starting point:

<AccordionGroup>
  <Accordion title="Customer lookup">
    Find customer by email `tom@example.com` (or `source_id`, or `customer_id`). Return the ID, `loyalty_balance`, `active_vouchers`.
  </Accordion>
  <Accordion title="Segment analysis">
    Count total of customers in segment "VIP". List their basic details: name, email address, `source_id`. Turn the data into a CSV-friendly format.
  </Accordion>
  <Accordion title="List campaigns">
    List active campaigns with fields: ID, name, type, `start_date`, `end_date`.
  </Accordion>
  <Accordion title="Voucher details">
    Get voucher by code "BK-4829" and show: status, `redemption.count`, `redemption.limit`, `balance` (for gift or loyalty cards).
  </Accordion>
  <Accordion title="Campaign data">
    Get campaign "BK-Sept-20OFF" data: total budget, spent budget, redemption counts, and per-customer caps.
  </Accordion>
  <Accordion title="Top coupon campaign">
    Show the campaign with the most coupons generated. Return redemption data for this campaign.
  </Accordion>
  <Accordion title="Best performing campaign">
    Show me the best performing campaign in terms of number of successful redemptions. Return the budget - the total discount value that was applied.
  </Accordion>
  <Accordion title="Redemption aggregation">
    Get redemptions aggregated by day between 2025-09-01 and 2025-09-03 (timezone Europe/Warsaw).
  </Accordion>
  <Accordion title="Best deals with cart">
    Get best deals for a customer with this email address. They have these items in their cart: Voucherify T-shirt (SKU: VCH-TST-001, quantity: 1, price: 25 USD), Voucherify Mug (SKU: VCH-MUG-002, quantity: 2, price: 15 USD each). Suggest if there's anything they can do to get even better deals.
  </Accordion>
</AccordionGroup>
