Fraud in loyalty and promotion systems usually falls into two categories: promotion fraud and identity fraud. Voucherify helps you prevent promotion fraud using built-in campaign controls. Identity fraud requires external tools.

## Types of fraud in loyalty programs

Understanding the difference between fraud types helps you choose the right prevention strategy.

### Promotion fraud

Promotion fraud happens when users misuse incentives such as vouchers, discounts, or loyalty points.

Examples include:
- sharing or reusing voucher codes  
- exceeding redemption limits  
- creating multiple accounts to reuse promotions  

### Identity fraud

Identity fraud happens when users manipulate or take over accounts.

Examples include:
- creating multiple accounts using fake or disposable emails  
- account takeovers  
- using internal or employee credentials as customer accounts  

## Campaign security

Use built-in controls to limit how incentives are distributed and redeemed.

Voucherify supports fraud prevention through:
- validation rules  
- redemption limits  
- activity tracking  

## Anti-fraud limits

Use validation rules to define limits and conditions for incentive usage.

<AccordionGroup>

<Accordion title="Code and customer controls">

Control who can receive and redeem incentives.

- use **Customer segment** to restrict access to selected users  
- use **Customer loyalty tier** to limit eligibility  
- use **Redemption only by code holder** to prevent sharing of codes  

</Accordion>

<Accordion title="Redemption limits">

Limit how often incentives can be redeemed using rules such as:

- **Total number of redemptions**  
- **Redemptions per customer in a campaign**  
- **Redemptions per customer per incentive**  

You can also apply time-based limits:
- **Total number of redemptions per day / week / month**  
- **Redemptions per customer in a campaign per day / week / month**  

</Accordion>

<Accordion title="Budget limits">

Control campaign spending using rules such as:

- **Total discounted amount**  
- **Total orders value**  
- **Total redeemed gift amount**  

</Accordion>

<Accordion title="Order and product conditions">

Define when an incentive can be applied based on order structure.

You can use rules such as:
- **Total amount before discounts**  
- **Total amount after discounts**  
- **Items quantity**  
- **Price of any item**  
- **Price of each item**  

You can also control which items qualify:
- **Any order item**  
- **Every order item**  
- **None of the order items**  
- **Most expensive of the order items**  
- **Cheapest of the order items**  

</Accordion>

<Accordion title="API and redemption controls">

Restrict how incentives are redeemed.

- use **Redeeming API key**  
- use **Redeeming user**  

<Info>
Rate limiting per IP must be implemented on your backend or API gateway.
</Info>

</Accordion>

<Accordion title="Time limits">

Restrict when incentives are valid.

You can define:
- campaign start and expiration dates  
- specific time windows  
- validity period after publication  

<Info>
Learn how to configure [time limits](/optimize/time-limits)
</Info>

</Accordion>

<Accordion title="Redemption rollback">

Undo redemptions when needed.

- restore redemption limits  
- restore balances such as gift cards or loyalty points  

<Warning>
Rollback is available up to 3 months after redemption.
</Warning>

</Accordion>

</AccordionGroup>

## Activity tracking

Monitor campaign activity to detect unusual patterns.

Voucherify lets you track:
- campaign performance  
- customer behavior  
- redemption history  
- API requests  

Each object (campaign, voucher, customer, order) includes logs and history in the dashboard.

## Notifications and reports

Use notifications and reports to detect issues early.

### Webhooks

Use webhooks to receive real-time notifications about events such as redemptions or customer updates. This lets you detect unusual activity and react as it happens. You can configure webhooks in [Project settings webhooks](/api-reference/project-settings-webhooks). For an overview, see [Introduction to webhooks](/api-reference/introduction-to-webhooks).

Recommended events for fraud monitoring:
- **redemption.succeeded** — track sudden spikes in usage  
- **redemption.failed** — detect repeated failed attempts  
- **voucher.updated** — monitor changes to incentives  
- **customer.updated** — track suspicious customer activity  

### Dashboard reports

Use dashboard reports to identify unusual patterns.

Focus on:
- spikes in redemptions  
- high failure rates  
- unexpected changes in campaign performance

## Recommended fraud prevention practices

These practices are not provided by Voucherify. You can implement them in your authentication or backend systems.

### Account protection

Protect user accounts from unauthorized access.

- limit failed login attempts  
- use CAPTCHA  
- enable multi-factor authentication  

### Customer verification

Reduce fake accounts and improve data quality.

- block suspicious email domains  
- normalize email addresses  
- require email or SMS verification
