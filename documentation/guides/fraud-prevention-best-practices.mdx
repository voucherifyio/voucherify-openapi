## Fraud prevention best practices

This guide explains how to secure campaigns, reduce abuse, and keep redemptions a reliable measure of success.

## Account security

Protect your account from unauthorized access.

- Enable two-factor authentication (2FA)
- Enable SAML authentication (if available)
- Use strong, unique passwords
- Change passwords regularly

## IP whitelisting

**Enterprise feature (available upon request)**

This feature lets you send API requests only from selected IP addresses.

To configure:
1. Go to **Project Settings → General → Application Keys**
2. Edit a key
3. Add allowed IP addresses

The allowlist is empty by default, so all IP addresses can send requests to the API.

## Campaign security

Control how incentives, such as vouchers and gift cards, are distributed and redeemed.

Voucherify supports fraud prevention through:
- anti-fraud limits,
- validation rules,
- logs, notifications, and reports.

## Anti-fraud limits

Use limits to control distribution and redemption of incentives.

<Accordion title="Code security">
Control how voucher codes are generated and used.

- Use unique, one-off codes
- Define code pattern, charset, and length
- Add prefix or postfix if needed
- Code case sensitivity is configured in project settings
</Accordion>

<Accordion title="Customer limits">
Control how a customer participates in a campaign.

- A customer can join a campaign only once; repeated requests return the same assigned code
</Accordion>

<Accordion title="Redemption limits">
Control how often incentives can be redeemed.

You can limit:
- redemptions per campaign
- redemptions per code
- redemptions per customer
- number of redeemables per order

When a limit is reached, further redemption attempts are rejected.
</Accordion>

<Accordion title="Budget limits">
Protect campaign budgets from overuse.

You can set:
- total discounted amount per campaign
- total order volume
- total redeemed gift card credits per campaign
</Accordion>

<Accordion title="Validation rules">
Define additional conditions for applying an incentive.

You can define:
- order conditions (e.g. minimum value)
- product-related conditions
- customer segments
- custom metadata
</Accordion>

<Accordion title="API and redemption controls">
Control who can redeem incentives via the API.

You can:
- limit redemptions to selected API keys or users
- use domain whitelisting for client-side requests
- apply rate limits per IP
</Accordion>

<Accordion title="Time limits">
Restrict when incentives can be redeemed.

You can define:
- start and expiration dates
- specific time windows (days or hours)
- validity period after publication (e.g. 48 hours after assignment)
</Accordion>

<Accordion title="Redemption rollback">
Rollback redemptions when needed.

You can:
- restore used redemption limits
- restore balances affected by redemption (e.g. gift card credits, loyalty points)

Rollback is available up to 3 months back.
</Accordion>

## Activity tracking

Monitor activity to detect suspicious behavior.

Voucherify lets you track:
- campaign performance (usage trends, spikes)
- customer behavior (unusual redemption patterns)
- redemption history (frequency, success rates)
- distribution results (delivery and usage of codes)

Each object (e.g. campaign, voucher, customer, order) has a dashboard with details, change history, and logs. The Activity tab records all related events.

### Logs

Inspect API activity using logs.

Logs include:
- request source (IP, API key)
- request status
- related entities (e.g. campaign, voucher, customer)

Logs are available in the dashboard and can be exported for analysis.

## Notifications and reports

Stay informed about campaign activity.

### Webhooks

Receive automatic notifications about system events.

Examples include:
- failed redemptions
- message delivery issues
- campaign updates

You can configure many event types.

### Dashboard reports

Monitor performance using dashboard reports.

You can track:
- total redemptions
- failure rates
- recent activity
- campaign results

### Distribution analytics

Analyze message delivery and distribution performance.

You can see:
- sent messages
- delivery status

## Additional fraud protection

Add extra safeguards against abuse.

### Failed attempts limits

Limit repeated failed redemption attempts.

You can:
- block users after too many failed attempts
- temporarily suspend access

### Customer verification

Reduce fraud and fake accounts.

You can:
- block email aliases
- normalize emails (e.g. lowercase)
- block suspicious domains
- require email or SMS verification

## External fraud prevention tools

Use third-party tools for advanced fraud detection. These tools are not built into Voucherify and require separate integration.

<Accordion title="Sift">
Machine learning fraud detection platform that analyzes user behavior in real time and helps detect suspicious activity.
</Accordion>

<Accordion title="SEON">
Fraud prevention tool that helps identify fake accounts and reduce abuse using data analysis and risk scoring.
</Accordion>

<Accordion title="Nethone">
AI-based solution that detects and blocks fraud before it happens by analyzing user behavior and device data.
</Accordion>

<Accordion title="FingerprintJS">
Visitor identification API that helps recognize users and detect suspicious patterns across sessions.
</Accordion>
