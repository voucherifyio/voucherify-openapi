## Fraud prevention best practices

Learn how to prevent promotion fraud and protect your campaigns using built-in features, validation rules, and external tools. This guide helps you make redemptions a reliable measure of success.

## Account security

Start by protecting your account from unauthorized access.

To improve security:
- Enable two-factor authentication (2FA)
- Enable SAML authentication (if available)
- Use strong and unique passwords
- Update passwords regularly

## IP whitelisting

Use IP whitelisting to control who can access your API.

**Enterprise feature (available upon request)**

This feature allows API requests only from selected IP addresses.

To configure:
1. Go to **Project Settings → General → Application Keys**
2. Edit a key
3. Add allowed IP addresses

If the list is empty, access is allowed from any IP.

## Campaign security

Protect your campaigns by controlling how promotions are used.

Voucherify supports fraud prevention through:
- anti-fraud limits,
- validation rules,
- tracking tools.

These features help reduce misuse and protect your budget.

## Anti-fraud limits

Use anti-fraud limits to control how promotions are distributed and redeemed. Expand the sections below to see available options.

<Accordion title="Code security">
Control how codes are generated and used.

- Use unique, one-off codes
- Define code pattern, charset, and length
- Add prefix or postfix if needed
- Codes are case-sensitive by default
</Accordion>

<Accordion title="Customer limits">
Limit how often a customer can use a campaign.

- A customer can join a campaign only once
- The same code is returned if requested again
</Accordion>

<Accordion title="Redemption limits">
Control how many times promotions can be used.

You can limit:
- total redemptions per campaign
- redemptions per code
- redemptions per customer
- redemptions per order

Example:
- A customer can redeem a welcome discount only once
- A code can be used only 50 times
</Accordion>

<Accordion title="Budget limits">
Protect your campaign budget from overuse.

You can set:
- total discount amount
- total order volume
- total gift card spend
</Accordion>

<Accordion title="Validation rules">
Define when and how a promotion can be used.

You can use:
- order conditions (e.g. minimum value)
- product requirements
- customer segments
- custom metadata

Example:
- Discount applies only to orders above $150
- Offer is available only for selected customers
</Accordion>

<Accordion title="API and access limits">
Control who can access and redeem promotions.

You can:
- limit redemptions to selected API keys or users
- use domain whitelisting for client-side requests
- apply rate limits per IP
</Accordion>

<Accordion title="Time limits">
Restrict when promotions are active.

You can define:
- start and expiration date
- specific days or hours
- validity after publication (e.g. 48 hours)
</Accordion>

<Accordion title="Redemption rollback">
Undo redemptions when needed.

You can:
- restore usage limits
- return gift card balance

Rollback is available up to 3 months back.
</Accordion>

## Activity tracking

Track all promotion activity to detect suspicious behavior.

Voucherify lets you monitor:
- campaign performance
- customer behavior
- redemption history
- distribution results

Each campaign, customer, and order has a detailed view with logs.

### Logs

Use logs to inspect all API activity.

Logs show:
- request source
- status
- related objects

They help with debugging and fraud detection.

### Filtering and export

Use filters and exports to analyze your data.

You can:
- filter vouchers and redemptions
- deactivate suspicious codes
- export data to CSV

## Notifications and reports

Stay informed about important events in your campaigns.

### Webhooks

Use webhooks to receive automatic notifications.

You can track events such as:
- failed redemptions
- message delivery issues
- campaign updates

### Dashboard reports

Use dashboard reports to monitor performance.

You can track:
- total redemptions
- failure rates
- recent activity
- campaign results

### Distribution analytics

Analyze how your messages and codes perform.

You can see:
- sent messages
- delivery status
- redemption rates

## Additional protective layers

Add extra protection on top of built-in features.

### Failed attempts limits

Limit repeated failed redemption attempts.

You can:
- block users after too many failed attempts
- temporarily suspend access

This helps prevent brute-force attacks.

### Customer verification

Verify users to reduce fraud and fake accounts.

You can:
- block email aliases
- normalize emails (e.g. lowercase)
- block suspicious domains
- require email or SMS verification
