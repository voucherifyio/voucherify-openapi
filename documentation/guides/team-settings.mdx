Team settings control your whole Voucherify account. Only the **account owner** can open and manage Team settings.

To open Team settings, open the profile menu in the bottom-left corner of the dashboard and select **Team settings**.

## Subscription

The Subscription section shows your plan, billing details, and usage.

You can:
- create a subscription with **Create subscription**, select a plan, confirm billing details, and add a payment card
- update or cancel your subscription
- update your payment method
- edit billing details in the Billing details section
- review usage and limits

Your billing cycle starts on the day you confirm your subscription. Usage limits reset at the start of each cycle.

If you exceed your usage limits, API access is blocked until the next cycle starts or you upgrade your plan.

### Usage overview

Team settings show usage across projects (API, messages, and webhooks). Use the project selector (the down arrow) to select a project and view its usage.

You can set notifications for usage (API, messages, and webhooks). Under the charts, select **Set notifications** to define:
- the threshold that triggers a notification (percentage of your limit)
- notification channel (email or in-app notification in the dashboard)
- recipients

For project-level usage details, go to **Project settings > API usage** in a selected project.

## Security

The Security section lets you enforce account-level security rules and check if all users are enrolled.

You can enforce two-factor authentication (2FA) for your team. Voucherify supports:
- SMS codes
- Google Authenticator
- backup codes

## Invoices

Voucherify issues invoices based on your subscription (monthly or annual). You can find them in the Invoices tab and download them.

## Team

The Team section is where you manage account users.

You can:
- use filters to find team members
- invite a new user with **Invite member**
- edit or remove a user using the three-dots (actions) menu

Removing a user is a two-step process. First delete the user. Then use the actions menu again to delete them permanently.

When you invite a user, select the projects they can access and assign a role for each project.

## Roles

Roles control what users can see and do in the dashboard and API. You can also review permissions for each role in this section.

### Owner flag

Owner is a flag on a user account that gives access to Team settings. Ownership can be granted only by Voucherify support on request from a current owner. It is recommended to have more than one owner.

### Built-in roles

When assigning a role to a user in a project, you can select from built-in roles:
- **Admin**: full access to projects and Project settings (cannot access Team settings)
- **User**: can read and change data in assigned projects
- **Viewer**: can read and export data, but cannot change it
- **Merchant**: limited access for redeeming incentives
- **Restricted user**: required when using areas and stores; access is limited to assigned areas or stores configured in Project settings

### Custom roles (enterprise)

On enterprise plans, you can create custom roles. Custom roles are defined for the whole account and assigned to users per project.

To manage roles, go to **Team settings > Roles**:
- add a new role by duplicating an existing role
- edit custom roles (built-in roles cannot be edited)

## Projects

The Projects section lists all projects in your account. You can view and filter projects.

Data is not shared between projects. Each project has separate customers, campaigns, and configuration.

You can:
- create a new project with **Create project** (if your subscription limits allow it)
- switch between projects using the actions menu to start working in another project
- delete a project using the actions menu (you must be in a different project to delete it)

### Create a project

When you create a project, you set:
- name
- description (optional)
- region (Europe, United States, or Asia (Singapore))
- time zone
- currency
- default international dialing prefix (optional)

You also set voucher code case sensitivity as a permanent setting:
- case sensitive: `cOdE1` and `CODE1` are different codes
- case insensitive: `cOdE1` and `CODE1` are the same code

<Warning>
You cannot change voucher code case sensitivity after you create the project.
</Warning>

During project creation, you must assign users and define their roles.

## Legal & compliance

Voucherify is a GDPR-compliant data processor. In Legal & compliance, you can:
- review policies, subscription agreements, and data processing agreements
- review and accept updates to policies
- add contact details such as:
  - EU representative
  - data protection officer
  - security contact person
- access compliance documents available in the dashboard

## Delete people data

Use Delete people data to permanently remove customer history for GDPR requests.

Search for a customer using identifiers such as name or source_id, then delete and confirm.

## Management API (enterprise)

The Management API is available on enterprise plans. It provides endpoints to manage:
- projects
- users
- schemas
- webhooks
- branding

You can review Management API usage in the Subscription section.
