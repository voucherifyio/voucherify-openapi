<Note>

This page describes changes to how you manage **Stacking rules** and **Categories** in Voucherify.

**The behavior of stacking rules doesn't change. You don't need to update your configuration. Stacking incentives work the same as before.**

There is a **Stacking rules** section in the **Campaign hub**. In this section, you can:
- view and manage campaign categories  
- assign campaigns to categories  
- set **Stacking settings**

![View of stacking rules showing existing categories, their hierarchies, and campaigns assigned to the categories. The view also includes a no category group that includes campaigns without any category assigned to them](/images/docs/stacking-rules-categories-view.png "Stacking rules view with categories")

![View of stacking settings showing application and sorting settings. The image shows also other group of settings: Discount behavior and global limits](/images/docs/stacking-rules-settings-view.png "Stacking settings view with Application and sorting rules")

Also, some stacking rule settings have new names.

<Accordion title="Stacking rules: name change list">

| Current setting name | New setting name |
| --- | --- |
| Discounts' application rules | Application and sorting |
| Application rule<br/>- Partial - inapplicable (not met criteria) discounts are omitted<br/>- All provided promotions must be in a valid state | Redemption stack validation policy<br/>- Partial<br/>- All |
| Sorting rule <br/>- Promotions applied in an order given in request<br/>- Promotions' calculation order defined by category hierarchy | Incentive application order<br/>- Requested order<br/>- Hierarchy priority |
| Sorting rule for grouped redeemables<br/>- Apply Joint categories last<br/>- Apply Sorting rule to combined Joint and Exclusive categories | Always stackable/Never stackable category application order<br/>- Apply Always stackable categories last<br/>- Apply Sorting rule to combined Always stackable and Never stackable categories |
| Products application rule<br/>- Always stack - allow multiple discounts to a product<br/>- Apply only once - prevent multiple discounts to the same product | Product stack policy<br/>- Multiple application<br/>- Single application |
| Rollback Order Mode <br/>- With order - The discounts will be reverted and order will be canceled<br/>- Without order - The discounts and status will be kept | Redemption rollback policy<br/>- Revert<br/>- Keep |
| No Effect Rule<br/>- Redeem anyway - even where there is no effect applied<br/>- Skip - redeem only when there is an effect applied | No effect<br/>- Force redemption<br/>- Prevent redemption |
| Limit Settings for Stackable Discounts API | Global limits |
| Maximum count of redeemables in API request | Maximum number of incentives in a request |
| Maximum count of valid promotions to apply in a request | Maximum number of valid incentives to apply in a request |
| Maximum count of valid promotions per category to apply in a request | Maximum number of valid incentives per category to apply in a request |
| Maximum count of valid exclusive promotions to apply in a request | Maximum number of valid Never stackable incentives to apply in a request |
| Maximum count of valid exclusive promotions per category to apply in a request | Maximum number of valid Never stackable incentives per category to apply in a request |
| Campaigns Stacking Eligibility<br/>- Never Stackable (Exclusive)<br/>- Always Stackable (Joint) | Stacking eligibility (*in the Category editor*)<br/>- Common<br/>- Never stackable<br/>- Always stackable |
| Discount Calculations by Category<br/>- Based on Initial Amount<br/>- Based on Discounted Amount | Discount calculation mode<br/>- Based on discounted amount<br/>- Based on initial amount |

</Accordion>
