Voucherify provides analytics tools that help you monitor promotional performance and system usage.

You can review data in three main places:

- **Dashboard** (project-level overview)
- **Team Settings** (organization-level usage analytics)
- **Project Settings** (project-specific usage analytics)

This article describes project-level and technical analytics. Campaign-level analytics are covered in a separate article.

## Dashboard overview

The Home dashboard shows a summary of activity in your project.

You can select a time window in the top right corner:

- Last 7 days  
- Last 30 days  
- Last 90 days  

All metrics and charts update based on the selected period.

### Key metrics

At the top of the dashboard, you can see four main indicators.

**Total redemptions**: The total number of successful voucher and promotion redemptions in the selected period.

**Revenue impact**: The total discount value applied to orders.

**Active campaigns**: The number of campaigns that are currently running.

**Daily API calls**: The total number of API requests in the selected period.

These indicators help you quickly understand promotional activity and overall platform usage.

### Redemption trends

The Redemption Trends chart shows how redemptions change over time.

It displays:

- Succeeded redemptions  
- Failed redemptions  

This allows you to compare success and failure rates during the selected timeframe.

### Top campaigns

The Top Campaigns table shows campaigns with the highest number of redemptions.

The table includes:

- Campaign name  
- Number of redemptions  
- Success rate  
- Trend indicator  

The trend indicator shows the performance direction.

If you click a campaign in this table, you will be redirected to its campaign dashboard.

### Insights and alerts

The Insights & Alerts section displays important notifications related to your campaigns.

For example, you may see that the campaign is expiring soon  

Each alert includes basic information and a direct action button, such as reviewing or extending a campaign.

This section helps you react quickly to important changes.

## Team and Project Settings analytics

In addition to the Dashboard, you can monitor technical usage analytics in **Team Settings** and **Project Settings**.

**Team Settings** display organization-wide analytics across all projects.  
**Project Settings** focus on analytics for the current project you are working in.

Some limits may appear in both places if they apply to the entire organization as well as to individual projects.

These sections help you track API traffic, limits, webhooks, and messaging activity.

## Team Settings

Team Settings provide **organization-level analytics across all projects** in your account.

Only the account owner can view and manage Team Settings.

### Management API calls per hour

This chart shows the number of management API calls per hour across the entire account.

It helps you monitor overall API activity.

### API calls per hour in a project

This chart shows API calls per hour for selected projects.

It also displays the API limit defined in your plan.

### API calls per subscription cycle

This chart shows the total number of API calls used during the current subscription cycle.

It helps you track usage within your billing period.

### Integration webhook calls per day

This chart shows webhook calls generated by integrations connected to Voucherify.

### Messages per day and per month

These charts show the number of messages sent per day and per month across projects.

## Project Settings

Project Settings provide analytics for the **current project environment**.

By default, the account owner can access Project Settings. Users with proper access granted by the owner, such as users assigned the default Admin role, can also view and manage Project Settings.

In the **API usage** tab, you can monitor:

- API calls per hour in the current project  
- API calls during the subscription cycle  
- Integration webhook calls per day  
- Messages per day  
- Messages per month  

Project-level analytics help you understand usage and limits within a specific project.

Some charts may still reference organization-level limits when those limits also apply to the project.
