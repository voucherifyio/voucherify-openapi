When customers use more than one incentive in a single order, stacking rules and categories define what happens. They control application order, compatibility between campaigns, discount calculation, and limits on how many incentives apply.

## Key concepts

- **Categories** group campaigns and define stacking eligibility: **Common**, **Never stackable**, or **Always stackable**.
- **Stacking rules** set global behavior for all redemptions: application order, validation policy, discount calculation, and limits.
- **Hierarchy** controls priority when **Incentive application order** is set to **Hierarchy priority**. A lower number means higher priority.

## How categories interact

| Category A | Category B | Result |
|---|---|---|
| Common | Common | Stack together |
| Common | Never stackable | Do not stack |
| Common | Always stackable | Stack together |
| Never stackable | Never stackable | Stack together |
| Never stackable | Always stackable | Stack together |
| Always stackable | Always stackable | Stack together |

Use **Compatibility guide** in the **Stacking rules** view to check how your categories interact before assigning campaigns.

## Why stacking rules matter

Without stacking rules, multiple incentives can combine in unexpected ways, leading to excessive discounts or conflicting promotions. Stacking rules help you:

- **Prevent over-discounting**: Set limits on how many incentives apply per redemption.
- **Protect exclusive offers**: Use **Never stackable** categories to block other campaigns from stacking with VIP or high-value promotions.
- **Control discount order**: Choose whether incentives apply in request order or category hierarchy order, which affects the final discount amount.
- **Handle edge cases**: Define what happens when an incentive has no effect or when a redemption is rolled back.
