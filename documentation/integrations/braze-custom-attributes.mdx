## Concept

This integration connects the Voucherify Incentive Optimization Engine with Braze as your engagement layer. Voucherify handles the optimization logic, including eligibility validation, guardrails, and real-time decisioning. Braze handles segmentation, journey orchestration, and message delivery.

You can trigger validated incentives from Voucherify and push them to Braze custom attributes. These attributes then become available for use in your Braze messaging experiments.

Supported incentive types:
- Discount coupons
- Gift cards
- Loyalty cards
- Referral codes
- Promotion tiers

## Logic

You can trigger validated incentives to Braze using:
- **Manual messages**: A one-time action that assigns validated codes or promotion tiers to a chosen audience (Voucherify segment or single customer).
- **Automated workflows**: Triggers a validated incentive in response to customer actions.

<Accordion title="Triggers that can push validated incentives to Braze">

    Voucherify can push codes or promotion tiers to Braze automatically in response to the following triggers:
    
    - **Customer entered/left segment** in Voucherify.
    - **Successfully published**: Triggered when a code from an incentive configuration is published (assigned) to a customer.
    - **Order status changed**: Triggered when an order changes status to created, updated, paid, or canceled.
    - **Reward redemption**: Triggered when a customer redeems a loyalty or referral reward.
    - **Custom event**: Triggered once Voucherify logs a specific custom event.
    - **Loyalty points adjusted**: Triggered when loyalty points are added to or subtracted from a customer profile.
    - **Customer earned loyalty points**: Triggered when a customer earns points through an earning rule.

</Accordion>

If a customer in Voucherify does not exist in Braze, the distribution creates a new user profile in Braze. The Voucherify customer `source_id` becomes the Braze `user_id`.

## Integration

In Voucherify, go to **Engagement** > **Distributions** to **Create distribution**.

<Steps>

  <Step title="Provide name and select trigger">

    Name your distribution and select the trigger.

    For a relevant trigger, select required options:
    - **Incentive configuration** (formerly Campaign)
    - **Segment**
    - **Validation rule**
    - **Custom event**

  </Step>

  <Step title="Choose action">

    - **Notify customers about promotion**: Select the **Promotion tier**. Promotion tiers are not individually assigned like codes. They notify customers of a promotion that will be applied automatically during checkout. Voucherify stores the promotion details in Braze customer profiles, including the environment and promotion code in three formats (text, barcode, QR).

    - **Trigger validated incentive from configuration**: Select the incentive configuration that will be the source of unique codes.

  </Step>

  <Step title="Configure channels">

    In **Channels**, select Braze. Map custom fields to the user profile in Braze:
    - For vouchers, map **Voucher code** to represent the `code` of the published voucher, for example `voucherify_incentive_code`. Other fields are optional.
    - For promotion tiers, map **Promotion tier name** and **Promotion tier ID**. Other fields are optional.

    <Tip>

    This step defines how Voucherify properties appear as custom attributes in Braze. You do not need to create these attributes in Braze beforehand. They map automatically to the defined key.

    </Tip>

  </Step>

  <Step title="Review and send">

    In **Summary**, verify the configuration. Select **Save and send** to deliver the validated incentives to user profiles in Braze.
    
  </Step>

</Steps>

Manual distributions are sent after a 10-minute delay. You can edit distribution details during this window.

Automated distributions start immediately.

Each message assigns a code or promotion tier as a custom attribute to a user profile in Braze.

## Orchestration

Once the custom attribute with the code is added to user profiles in Braze, you can use it in your Braze messaging experiments.

Edit the message body and add the custom attribute defined in the Voucherify distribution (for example, `voucherify_incentive_code`). To display the unique code, use the liquid code `{{custom_attribute.${voucherify_incentive_code}}}`.

You can preview the code in your message for a specific user.

## Optimization

Use Voucherify guardrails to protect margins during experimentation:

- **Validation rules**: Define eligibility criteria that must be met before any incentive is triggered.
- **Usage limits**: Set caps on how many times a code can be redeemed per customer or globally.
- **Time-based constraints**: Control when incentives are valid with start dates, expiration dates, and time windows.
- **Budget controls**: Set spending limits to protect against over-discounting.

These guardrails ensure that every incentive pushed to Braze has already passed real-time validation in Voucherify.
