## Concept

This integration enables continuous incentive experimentation in Braze by pushing validated incentives from Voucherify to custom attributes in Braze user profiles. Your Incentive Optimization Engine (Voucherify) handles eligibility validation, guardrails, and real-time decisioning. Your Engagement Layer (Braze) handles segmentation, delivery, and messaging.

You can trigger the following incentive types:
- Discount coupons
- Gift cards
- Loyalty cards
- Referral codes
- Promotion tiers

## Logic

Voucherify acts as the Optimization Layer that validates and enforces:

- **Eligibility rules**: Determine which customers qualify for specific incentives
- **Guardrails**: Margin protection limits like redemption caps, budget limits, and expiration logic
- **Real-time decisioning**: Cross-channel enforcement to prevent abuse

Braze acts as the Engagement Layer that handles:

- Audience segmentation
- Journey orchestration
- Message delivery and testing

<Accordion title="Experiment triggers">

Voucherify can trigger validated incentives to Braze in response to these events:

- **Customer entered/left segment**: Run experiments when customers join or leave a Voucherify segment
- **Successfully published**: Trigger incentive when a code is assigned to a customer in Voucherify
- **Order status changed**: Respond to order lifecycle events (created, updated, paid, canceled)
- **Reward redemption**: Trigger when a customer redeems a loyalty or referral reward
- **Custom event**: Trigger based on any custom event logged in Voucherify
- **Loyalty points adjusted**: Trigger when loyalty points change on a customer profile
- **Customer earned loyalty points**: Trigger when a customer earns points through an earning rule

</Accordion>

If a customer exists in Voucherify but not in Braze, the distribution creates a new Braze user profile using the Voucherify customer `source_id` as the Braze `user_id`.

## Integration

Configure incentive distribution in Voucherify to push validated incentives to Braze custom attributes.

<Steps>

<Step title="Select experiment trigger">

In Voucherify, go to **Engagement** > **Distributions** and select **Create distribution**.

Name your distribution and select the trigger that starts the experiment. Based on your trigger, configure:
- **Campaign**: The source of incentives
- **Segment**: Target audience for the experiment
- **Validation rule**: Guardrails and eligibility criteria
- **Custom event**: The event that triggers the experiment

</Step>

<Step title="Configure incentive action">

Select the incentive action:

- **Notify customers about promotion**: Select a promotion tier. Customers receive notification that a promotion applies automatically at checkout. This is useful for experiments with auto-applied discounts.

- **Send and publish unique codes from campaign**: Select the campaign that provides unique codes. Each customer receives a validated, personalized incentive code.

</Step>

<Step title="Map custom attributes">

In **Channels**, select Braze. Map Voucherify properties to Braze custom attributes:

For voucher codes:
- Map **Voucher code** to a custom attribute key (for example, `voucherify_campaign_code`)
- Additional fields are optional

For promotion tiers:
- Map **Promotion tier name** and **Promotion tier ID**
- Additional fields are optional

<Tip>

Voucherify creates these custom attributes in Braze automatically. You do not need to pre-configure them in Braze.

</Tip>

</Step>

<Step title="Run experiment">

Review your configuration in **Summary**. Select **Save and send** to trigger validated incentives to Braze user profiles.

- Manual distributions start after a 10-minute delay. You can edit the configuration during this window.
- Automated distributions start immediately when the trigger condition is met.

</Step>

</Steps>

## Orchestration

Use the custom attributes in your Braze campaigns and Canvases to deliver personalized incentives.

In your Braze message template, reference the custom attribute using Liquid:

```liquid
{{custom_attribute.${voucherify_campaign_code}}}
```

This displays the validated incentive code that passed through your guardrails and eligibility rules before reaching the customer.

Preview your message in Braze to verify the incentive code appears correctly for target users.

## Optimization

### Customer synchronization for segment-based experiments

To run experiments triggered by customer segments, synchronize your Braze users with Voucherify:

- Use [Braze Canvas synchronization](/integrations/send-data-from-braze#synchronize-customers-with-braze-canvas) for ongoing sync
- Import via [CSV in the Voucherify dashboard](/prepare/customers#csv-import)
- Import via the [Import and Update Customers API](/api-reference/customers/import-and-update-customers-using-csv)

When importing, map the Braze `user_id` to the Voucherify `source_id`.

### Segment configuration

Create [customer segments](/prepare/customer-segments) in Voucherify to target specific experiment audiences:

- Filter by `source_id` pattern (for example, `starts with braze`)
- Filter by `brazeAudiences` metadata containing Braze `audience_id` values

This enables targeted experimentation while maintaining guardrails and margin protection across all channels.
